## 1. 选型总览

### 1.1 技术栈全景

```
┌──────────────────────────────────────────────────────────────┐
│  层          │ 技术选型                 │ 语言          │
├──────────────────────────────────────────────────────────────┤
│  QML 层      │ Qt 6.8 LTS + Qt Quick   │ QML + JS      │
│  Model 层    │ Qt 6.8 Core + Qt QML    │ C++17         │
│  EventBus 层 │ 自研 (STL 实现)          │ 纯 C++ (仅 STL) │
│  Controller层│ 自研 (纯逻辑)             │ 纯 C (头文件)   │
│  Driver-I 层 │ 自研 (虚基类)            │ 纯 C++17       │
│  Driver 层   │ libssh2 + sqlite3 amalg. │ C              │
└──────────────────────────────────────────────────────────────┘
```

### 1.2 核心选型原则

1. **分层解耦**：上层不感知下层实现，通过接口隔离
2. **最小 Qt 依赖**：只有 QML 层和 Model 层可以使用 Qt，防止框架锁定
3. **C ABI 优先**：Driver-Interface 使用纯 C 头文件，支持未来替换为其他语言实现的驱动
4. **离线优先**：所有功能不依赖云服务
5. **性能优先**：本地同步模式下，sqlite3 C API 直接调用，零网络开销

## 2. Qt 框架层

### 2.1 Qt 版本

| 项目        | 选择                                    | 理由                      |
| --------- | ------------------------------------- | ----------------------- |
| **Qt 版本** | **Qt 6.8 LTS**                        | 最新 LTS，修复周期长，QML 性能大幅提升 |
| **许可证**   | LGPLv3 / GPLv3                        | 桌面应用，动态链接 LGPL          |
| **模块**    | Qt Quick, Qt Quick Controls 2, Qt QML | 核心 UI 所需                |

### 2.2 QML 层技术细节

| 需求    | 方案                                                                | 说明                                                    |
| ----- | ----------------------------------------------------------------- | ----------------------------------------------------- |
| UI 框架 | **Qt Quick + Qt Quick Controls 2**                                | 声明式 UI，适合拟态化风格                                        |

| 动画    | `Behavior` / `Transition` / `NumberAnimation`                     | `duration: 200-300ms`, `easing.type: Easing.OutCubic` |
| 主题系统  | **StyleSingleton**（QML Singleton）                                 | 深色/浅色 CSS 变量映射为 QML 只读属性                              |
| 表格    | `TableView` (Qt Quick)                                            | 虚拟化渲染，支持大数据量                                          |
| 代码编辑器 | **QSyntaxHighlighter** + 自研文本编辑组件                                 | 基于 `TextArea` / `TextEdit`，自定义 SQL 高亮规则               |
| 图标    | **Lucide Icons** (SVG → QML) 或 **Qt 内置图标**                        | 与参考项目视觉一致                                             |
| 字体    | `font.family: "Segoe UI"` / `"PingFang SC"` / `"Microsoft YaHei"` | 跨平台兜底                                                 |

#### 2.2.1 拟态化阴影 QML 实现方案

```qml
// 凸起阴影（卡片）
DropShadow {
    anchors.fill: parent
    source: parent
    radius: 8
    samples: 17
```

**替代方案**：使用 `Canvas` 绘制自定义阴影，或使用 `ShaderEffect` 实现双层阴影。

### 2.3 Model 层技术细节

| 组件                  | Qt 基类                    | 说明                            |
| ------------------- | ------------------------ | ----------------------------- |
| ConnectionListModel | `QAbstractListModel`     | SSH 连接列表                      |
| DatabaseTreeModel   | `QAbstractItemModel`     | 对象树（Table/View/Index/Trigger） |
| TableDataModel      | `QAbstractTableModel`    | 表数据，支持分页                      |
| SqlHistoryModel     | `QAbstractListModel`     | SQL 执行历史                      |
| SftpFileModel       | `QAbstractListModel`     | SFTP 远端文件列表                   |
| ThemeModel          | `QObject` + `Q_PROPERTY` | 主题配置单例                        |

**核心原则**：

- Model 层**不直接调用** EventBus，而是通过 `QMetaObject::invokeMethod` 将自身注册为 EventBus listener
- Model 层使用 Qt 的信号槽与 QML 层绑定
- 数据更新来自下层事件回调，Model 在回调中调用 `beginResetModel()` / `endResetModel()`

## 3. 纯 C/C++ 基础设施

### 3.1 EventBus 层

| 项目   | 选择                                       | 说明        |
| ---- | ---------------------------------------- | --------- |
| 语言   | **纯 C++17** (仅 STL)                      | 不依赖 Qt    |
| 线程模型 | **发布者-订阅者 (Pub/Sub)**                    | 支持同步+异步分发 |
| 事件存储 | `std::variant` + 类型标签                    | 类型安全的事件携带 |
| 线程安全 | `std::mutex` + `std::condition_variable` | —         |
| 内存管理 | `std::shared_ptr` / `std::unique_ptr`    | 明确所有权     |

#### EventBus 核心接口

```cpp
// eventbus/eventbus.h
class EventBus {
public:
    // 注册监听器
    template<typename E>
    ListenerId addListener(std::function<void(const E&)> handler);

    // 发送事件（同步，在当前线程执行所有回调）
    template<typename E>
    void emit(const E& event);

    // 发送事件（异步，放入队列由工作线程处理）
    template<typename E>
    void emitAsync(const E& event);

    // 移除监听
    void removeListener(ListenerId id);

    // 单例
    static EventBus& instance();
};
```

#### 事件类型体系

```cpp
// eventbus/events.h
struct SshConnectedEvent { std::string connectionId; };
struct SshDisconnectedEvent { std::string connectionId; std::string reason; };
struct SshErrorEvent { std::string connectionId; std::string message; int code; };

struct QueryResultEvent { std::string requestId; std::vector<std::vector<std::string>> rows; std::vector<std::string> columns; double elapsedMs; };
struct QueryErrorEvent { std::string requestId; std::string error; };

struct SftpTransferProgressEvent { std::string taskId; int64_t transferred; int64_t total; };
struct SftpTransferCompleteEvent { std::string taskId; std::string localPath; };
struct SftpTransferErrorEvent { std::string taskId; std::string error; };
// ... 更多事件
```

## 4. SSH / SFTP 驱动

### 4.1 SSH 库选型

| 候选             | 语言  | 协议   | 许可证  | 评价                              |
| -------------- | --- | ---- | ---- | ------------------------------- |
| **libssh2** ✅  | C   | SSH2 | BSD  | 成熟稳定，Curl 同款，内置 SFTP 子系统        |
| libssh         | C   | SSH2 | LGPL | 功能全但 LGPL 限制，CMake 集成不如 libssh2 |
| OpenSSH (进程调用) | —   | —    | —    | 依赖系统安装，可控性差，放弃                  |

**选择：libssh2**

理由：

- BSD 许可证，无分发限制
- 内置 SFTP 子系统（libssh2_sftp_*），一套 API 搞定 SSH + SFTP
- 广泛使用（curl、很多桌面应用），社区活跃
- C ABI，对 Driver-Interface 层友好
- 支持：密码认证、公钥认证、键盘交互、SSH Agent

### 4.2 Driver-Interface (SSH / SFTP)

```cpp
// driver_interface/issh_driver.h
struct SshConfig {
    std::string host;
    int port = 22;
    std::string username;
    enum AuthMethod { Password, PublicKey, SshAgent } authMethod;
    std::string password;          // AuthMethod == Password
    std::string privateKeyPath;    // AuthMethod == PublicKey
    std::string passphrase;        // 可选
    std::string jumpHost;          // ProxyJump 格式: user@host:port
    int connectTimeout = 30;       // 秒
    int keepAliveInterval = 60;    // 秒
};

class ISshDriver {
public:
    virtual ~ISshDriver() = default;

    // 连接管理
    virtual std::string connect(const SshConfig& config) = 0;  // 返回 connectionId
    virtual void disconnect(const std::string& connectionId) = 0;
    virtual bool isConnected(const std::string& connectionId) = 0;

    // 远程命令执行
    struct ExecResult {
        int exitCode;
        std::string stdout;
        std::string stderr;
    };
    virtual ExecResult execCommand(const std::string& connectionId,
                                   const std::string& command,
                                   int timeoutSec = 30) = 0;

    // SFTP 获取（快捷方法，复杂 SFTP 操作见 ISftpDriver）
    virtual std::unique_ptr<ISftpDriver> createSftpDriver(
        const std::string& connectionId) = 0;

    // 连接信息
    virtual std::string getHostKey(const std::string& connectionId) = 0;
};

// driver_interface/isftp_driver.h
class ISftpDriver {
public:
    virtual ~ISftpDriver() = default;

    virtual std::vector<FileEntry> listDirectory(
        const std::string& remotePath) = 0;

    virtual void download(const std::string& remotePath,
                          const std::string& localPath,
                          std::function<void(int64_t, int64_t)> progressCallback) = 0;

    virtual void upload(const std::string& localPath,
                        const std::string& remotePath,
                        std::function<void(int64_t, int64_t)> progressCallback) = 0;

    virtual bool fileExists(const std::string& remotePath) = 0;
    virtual int64_t getFileSize(const std::string& remotePath) = 0;
    virtual std::string getFileMtime(const std::string& remotePath) = 0;

    virtual void remove(const std::string& remotePath) = 0;
    virtual void mkdir(const std::string& remotePath) = 0;
};
```

### 4.3 Driver 实现 (libssh2)

```cpp
// driver/libssh2_ssh_driver.h
class LibSsh2SshDriver : public ISshDriver {
    // 封装 libssh2_session_*, libssh2_channel_* 等 C API
```

## 5. SQLite 驱动

### 5.1 远程 SQLite Driver（命令模式）

| 项目   | 选择                                        | 说明                   |
| ---- | ----------------------------------------- | -------------------- |
| 实现方式 | `ISshDriver::execCommand` + `sqlite3` CLI | 远端必须安装 sqlite3 命令行   |
| 输出格式 | `sqlite3 -json <db> "<sql>"`              | 结构化 JSON，易于解析        |
| 命令构建 | `RemoteSqliteCommandBuilder`              | 将操作翻译为 sqlite3 命令    |
| 解析   | **nlohmann/json**                         | 单头文件 JSON 解析库，MIT 协议 |

```cpp
// driver/remote_sqlite_driver.h
class RemoteSqliteDriver : public IRemoteSqliteDriver {
public:
    // 通过 ISshDriver 执行远程 sqlite3 命令
    RemoteSqliteDriver(std::shared_ptr<ISshDriver> ssh, const std::string& connectionId);

    QueryResult query(const std::string& dbPath, const std::string& sql) override;
    std::vector<TableInfo> getTables(const std::string& dbPath) override;
    std::vector<ColumnInfo> getTableInfo(const std::string& dbPath,
                                         const std::string& tableName) override;
    // ...
private:
    // 转义 SQL，构建 ssh exec 命令
    std::string buildSqlite3Command(const std::string& dbPath,
                                    const std::string& sql);
};
```

### 5.2 本地 SQLite Driver（SFTP 同步模式）

| 项目   | 选择                                                                      | 说明         |
| ---- | ----------------------------------------------------------------------- | ---------- |
| 实现方式 | **sqlite3 amalgamation** (sqlite3.c + sqlite3.h)                        | 单文件 C 源码嵌入 |
| API  | 原生 sqlite3 C API (`sqlite3_open`, `sqlite3_exec`, `sqlite3_prepare_v2`) | 高性能        |
| 编译   | 直接编译进项目，无需系统安装                                                          | 离线可用       |

```cpp
// driver/local_sqlite_driver.h
class LocalSqliteDriver : public ILocalSqliteDriver {
public:
    // 直接封装 sqlite3 C API
    bool open(const std::string& dbPath) override;
    void close() override;
    QueryResult query(const std::string& sql) override;
    bool execute(const std::string& sql) override;
    // ...
private:
    sqlite3* db_ = nullptr;
};
```

### 5.3 SQLite 版本要求

| 环境                      | 最低版本        | 说明                |
| ----------------------- | ----------- | ----------------- |
| 远程设备 sqlite3 CLI        | **3.37.2+** | 需要 `-json` 输出模式支持 |
| 本地 sqlite3 amalgamation | **3.45.0+** | 内嵌到应用，使用最新稳定版     |

## 6. 构建与打包

### 6.1 构建系统

| 项目   | 选择                                                    | 理由               |

### 6.2 CMake 项目结构（规划）

```
CMakeLists.txt                    # 顶层
cmake/
  FindLibSSH2.cmake               # 查找 libssh2
  QtSetup.cmake                   # Qt 配置
src/
  qml/CMakeLists.txt              # QML 资源
  model/CMakeLists.txt             # Model 层 (Qt 依赖)
  eventbus/CMakeLists.txt          # EventBus 层 (纯 STL)
  controller/CMakeLists.txt        # Controller 层 (纯 STL)
  driver_interface/CMakeLists.txt  # Driver 接口 (纯头文件)
  driver/CMakeLists.txt            # Driver 实现
```

#### 顶层 CMakeLists.txt 示例

```cmake
cmake_minimum_required(VERSION 3.27)
project(RemoteSQLite-Qt VERSION 0.1.0 LANGUAGES CXX C)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# Qt 6
find_package(Qt6 6.5 REQUIRED COMPONENTS Quick QuickControls2 Qml)

# 分层子目录
add_subdirectory(src/driver_interface)   # 纯头文件，INTERFACE library
add_subdirectory(src/eventbus)           # 纯 C++，STATIC library
add_subdirectory(src/controller)         # 纯 C++，STATIC library
add_subdirectory(src/driver)             # C/C++，STATIC library
add_subdirectory(src/model)              # C++ + Qt，STATIC library
add_subdirectory(src/qml)                # QML 资源 + main.cpp

# 主可执行文件
qt_add_executable(RemoteSQLite-Qt
    src/main.cpp
    ${QML_RESOURCES}
)

target_link_libraries(RemoteSQLite-Qt PRIVATE
    model_lib controller_lib eventbus_lib driver_lib
    Qt6::Quick Qt6::QuickControls2 Qt6::Qml
)
```

| 自动补全 | 内置 IntelliSense    | 需手写                         |
| 开发成本 | 低                  | 中                           |
| 启动速度 | 慢 (加载 WebEngine)   | 快                           |
| 推荐度  | ❌ 太重               | **✅ 首选**                    |

> **决策**：Qt 版本不使用 Monaco Editor（需嵌入 WebEngine，体积和内存开销过大），改用自研 QML SQL 编辑器 + QSyntaxHighlighter。SQL 关键字高亮规则可参考 Monaco 内置的 SQL 语法定义。
